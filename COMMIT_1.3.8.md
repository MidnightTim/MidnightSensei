# v1.3.8 — Shaman Pass, Spell/Talent Snapshots, Debug Overhaul & Level Gate

## Analytics

### feat(analytics): level 80 gate on fight recording
Fights below player level 80 are silently discarded — no history, no broadcast. Prevents leveling kills, old-world farming, and timewalking alts from polluting history and averages. Gate uses `UnitLevel("player")` immediately after the minimum fight duration check in `OnCombatEnd`.

## Spell & Talent Snapshot System

### feat(debug): persistent spell snapshot
`SPELLS_CHANGED` registered. On trigger (debounced 1s), `BuildSpellSnapshot()` enumerates the full spellbook via `C_SpellBook.GetSpellBookItemInfo` / `C_SpellBook.GetSpellBookSkillLineInfo` and writes `CharDB.spellSnapshot = { timestamp, specName, className, spells[] }`. Sorted by spellID.

### feat(debug): persistent talent snapshot
On `PLAYER_TALENT_UPDATE` / `PLAYER_SPECIALIZATION_CHANGED` (debounced 1s), `BuildTalentSnapshot()` walks `C_Traits` and writes `CharDB.talentSnapshot = { timestamp, specName, className, talents[] }`. Sorted by nodeID.

### feat(debug): snapshot export commands read from CharDB
`/ms debug talents` and `/ms debug spells` now read from the persisted snapshots rather than doing live API enumeration. Output includes capture timestamp and spec. If no snapshot exists, directs user to `/reload`.

### chore(debug): SESSION_READY builds both snapshots
2-second delay after login to ensure spellbook is fully loaded before enumeration.

## Grade History

### fix(ui): remove All Characters filter
Since schema v3 (1.3.2), encounters live in `SavedVariablesPerCharacter`. "All Characters" silently showed only current character data — identical to "This Character". Removed filter button and dead `"all"` branch from `FilterEncounters`. Reverting to account-wide storage would reintroduce the cross-character bleed that motivated the v3 split.
